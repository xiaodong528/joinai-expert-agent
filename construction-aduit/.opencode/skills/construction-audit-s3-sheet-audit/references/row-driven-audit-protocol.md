# Row-Driven Audit Protocol

用于 `construction-audit-s3-sheet-audit` 的确定性审计协议。

## 核心顺序

1. 从 `rule_doc.md` 中抽取当前 sheet 的规则表行。
2. 按原文顺序枚举费用名称，不得跳行。
3. 在目标 sheet 中先定位费用名称列，再定位对应业务行。
4. 对每个业务行按 `amount_role` 展开金额目标格。
5. 仅在目标格需要确定性数值复核时调用 `calc_formula.py`。

## 表一（451定额折前）样例模板

### 规则表行

必须按以下顺序枚举 9 项：

1. 建筑安装工程费
2. 需要安装的设备费
3. 工程建设其他费
4. 不需要安装的设备、工具费
5. 小型建筑工程费
6. 合计
7. 预备费
8. 总计
9. 企业运营费

### 行锚点

- 费用名称列固定先看 `C` 列
- 当前样例对应行为：
  - `C7` 建筑安装工程费
  - `C8` 需要安装的设备费
  - `C9` 工程建设其他费
  - `C10` 不需要安装的设备、工具费
  - `C11` 小型建筑工程费
  - `C12` 合计
  - `C13` 预备费
  - `C14` 总计
  - `C15` 企业运营费

### 目标格展开

- `amount_role=pre_tax` -> `J`
- `amount_role=tax` -> `K`
- `amount_role=tax_inclusive` -> `L`

样例中必须显式覆盖：

- 建筑安装工程费：`J7`、`K7`、`L7`
- 需要安装的设备费：`J8`、`K8`、`L8`
- 工程建设其他费：`J9`、`K9`、`L9`
- 合计：`J12`、`K12`、`L12`
- 预备费：`J13`、`K13`、`L13`
- 总计：`J14`、`K14`、`L14`

### 当前样例的来源映射

- 建筑安装工程费
  - `J7 <- 安全生产费计算表（表二折前）D6`
  - `K7 <- 安全生产费计算表（表二折前）D5 - D6`
  - `L7 <- J7 + K7`
- 需要安装的设备费
  - `J8 <- 表四-需安装设备 I19`
  - `K8 <- 表四-需安装设备 J19`
  - `L8 <- J8 + K8`
- 工程建设其他费
  - `J9 <- 表五（甲）D20`
  - `K9 <- 表五（甲）E20`
  - `L9 <- J9 + K9`
- 合计
  - `L12 <- J12 + K12`
- 预备费
  - `L13 <- J13 + K13`
- 总计
  - `L14 <- J14 + K14`

### 行类型

- `source_backed`
  - 建筑安装工程费
  - 需要安装的设备费
  - 工程建设其他费
- `not_applicable`
  - 不需要安装的设备、工具费
  - 小型建筑工程费
  - 企业运营费
- `derived_summary`
  - 合计
  - 预备费
  - 总计

## Findings 粒度

- findings 必须以“目标单元格”为粒度，不以“费用项”或“规则行”为粒度。
- `J/K` 只作为支撑校验，不直接产出 findings。
- 正式 findings 只从 `report_as_finding=true` 的 `L` 列目标格产出。
- 对 `L` 列报错时，`expected_value` 使用同一行 `J+K` 的结果，并按工作簿显示口径保留两位小数。
- 不能因为 `J7` 正确就默认 `L7` 也正确。
- `derived_summary` 行中的 `L` 列仍需校验；它们不能因为是派生汇总格就自动跳过。
