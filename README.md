# JAS Expert Agents

这是 JAS（JoinAI Swarm Factory）的专家智能体资产仓库，用来沉淀领域 Agent、阶段化 Skills，以及围绕多智能体协作形成的开发约束。

当前仓库聚焦两类已经形成能力资产的方向：
- `video-generation`：AI 视频生成专家智能体，覆盖从创意策划到最终 QA 的端到端多阶段管线。
- `construction-aduit`：工程审计专家智能体，覆盖规则文档解析、工作簿桥接、按表审核和报告生成。

下一步重点是建设“专家智能体开发智能体”，把新的专家智能体从设计、拆解、验证到接入 JAS 的过程进一步标准化。

## JAS 架构定位

JAS 采用 `OpenCode + Gas Town` 的双层架构：

- OpenCode 层负责专家知识表达。
  这里定义领域 Agent、Skills、执行约束，以及与具体业务强相关的工作流逻辑。
- Gas Town 层负责运行时基础设施。
  这里提供 Mayor / Polecat / Refinery / Witness 的编排、监控、恢复、任务派发和进度跟踪能力。

可以把它理解为：

- OpenCode 回答“做什么、怎么做”
- Gas Town 回答“谁来做、做到哪了”

这种拆分让专家智能体可以一边保留领域知识的高表达能力，一边复用稳定的多智能体运行时能力。

### 1. 视频生成专家智能体

`video-generation` 负责将 AI 视频制作流程拆成明确的阶段能力，并把适合并行的部分交给多智能体协同处理。

它当前覆盖的核心链路包括：
- 创意策划与 `story.yaml` 冻结
- 分镜脚本生成与角色锚定
- 关键帧生成、图生视频、TTS、BGM
- 视频拼接、字幕、对口型、最终 QA

这个方向的重点不是单个脚本，而是“如何把一个长链路、多模型、多模态的视频生成流程拆成可调度、可重试、可并行的技能单元”。

### 2. 工程审计专家智能体

`construction-aduit` 面向工程预算/结算审核场景，强调四角色协作和严格的阶段门控。

它当前覆盖的核心链路包括：
- 审核会话初始化与输入契约校验
- 规则文档渲染
- 工作簿桥接与结构化导出
- 单 Sheet 审核并行执行
- Findings 聚合与最终报告生成

这个方向的重点是“如何把高可靠、强约束、可追溯的审核流程交给多角色智能体协作执行”，尤其强调：
- Mayor 只负责编排和与用户交互
- Polecat 负责技能执行
- Refinery 负责质量审查
- Witness 负责运行监控与异常升级

## 仓库结构

当前仓库保留的是“专家能力定义层”，而不是完整运行时工程。

其中 `construction-aduit` 是当前仓库中的实际目录名，对应的业务语义是“工程审计 / 工程审核”方向。

```text
.
├── construction-aduit/
│   ├── agents/
│   └── skills/
└── video-generation/
    ├── agents/
    └── skills/
```

各层职责如下：

- `agents/`
  存放主编排智能体或角色智能体定义，描述职责、阶段边界、协作方式和执行约束。
- `skills/`
  存放阶段化能力定义，约定输入输出、执行步骤、校验清单和失败条件。

换句话说，这个仓库更像 JAS 的“专家能力资产目录”，而不是某个完整业务项目的运行入口。

## 如何新增一个专家智能体

如果要在 JAS 中新增一个领域专家智能体，推荐按下面的顺序推进：

1. 先定义领域边界。
   明确这个专家智能体要解决什么问题、输入输出是什么、成功标准是什么。
2. 再设计 Agent 形态。
   简单场景可以先从单编排 Agent 起步；复杂场景优先考虑四角色模型。
3. 把核心阶段沉淀成 Skills。
   每个 Skill 都应说明输入契约、输出契约、执行步骤、验证清单和失败条件。
4. 最后再对接 Gas Town。
   当领域逻辑稳定后，再把角色映射、任务派发、监控和恢复接入运行时层。

这个仓库的重点就是前三步：先把专家知识和执行结构沉淀下来，再进入更完整的运行时接入。

## 近期路线图

### 专家智能体开发智能体

仓库的下一阶段重点，不是再堆一个单独业务流程，而是建设“专家智能体开发智能体”。

它的目标是帮助团队更快、更一致地开发新的 JAS 专家智能体，包括：
- 帮助明确领域边界、角色划分和能力拆解
- 生成 Agent / Skill 的初始骨架与约束模板
- 给出从 OpenCode 资产到 Gas Town 接入的建议路径
- 将“已验证过的专家智能体开发流程”沉淀成可复用的方法论

当前它仍处于筹备阶段。README 在这里把它作为明确的方向提出，但不把它描述成已经完成的第三个模块。

## 总结

这个仓库承载的是 JAS 的专家能力资产层：
- 用 Agent 表达角色职责和协作结构
- 用 Skill 表达阶段能力和执行契约
- 用双层架构把领域知识和运行时能力解耦

当前已经沉淀视频生成与工程审计两个方向的专家能力资产，后续会继续补齐更多领域专家智能体，以及面向“专家智能体开发本身”的开发智能体能力。
