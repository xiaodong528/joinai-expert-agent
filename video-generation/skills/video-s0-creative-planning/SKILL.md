## story.yaml 完整 Schema

```yaml
# ── 故事大纲 ──
视频类型: ""          # 短剧 / 宣传片 / 教程 / 广告 / 创意短视频 / Vlog
类型风格: ""          # 都市 / 古装 / 科幻 / 悬疑 / 甜宠 / 搞笑 / 商务 / 科技
主线内容: |           # 剧情概括 / 产品卖点 / 教学要点，3-5 句
  ...
目标时长: ""          # 如 15秒 / 1分钟 / 3分钟 / 5分钟
单镜时长: "12秒"      # 默认 12秒，最大 12秒
目标画幅: ""          # 16:9（横屏）或 9:16（竖屏）

# ── 角色 DNA 卡片 ──
角色:
  - 姓名: ""
    性别: ""
    年龄段: ""              # 如 "25岁左右的青年"
    气质: ""                # 如 "冷峻内敛" / "开朗活泼"
    外貌描述:
      面部特征: ""          # 眼形、鼻型、标志性特征
      发型发色: ""
      肤色质感: ""
      体型: ""
    服装:
      默认服装: ""          # 详细描述
      备用服装: ""          # 如有换装场景
    标志配饰: ""            # 眼镜、项链、纹身等
    声音特征: ""            # 用于 TTS voice 选择
    情感范围: []            # 如 [平静, 愤怒, 悲伤, 惊讶]
    一致性锚定:
      LoRA_触发词: ""
      IP_Adapter_参考图: ""
      风格标签: ""
    禁止变更项: []          # 如 [发色, 标志配饰]

# ── 世界观与场景设定 ──
世界观与场景设定:
  时代背景: ""
  地理环境: ""
  主要场景:
    - 编号: 1
      名称: ""
      描述: ""
      氛围:
        光线: ""
        色温: ""
        情绪: ""

# ── 风格锁定 ──
风格锁定:
  画面风格: ""              # 电影级写实 / 日式动漫赛璐璐 / 3D皮克斯风
  色调: ""                  # 暖色调低饱和 / 冷色调高对比
  光照: ""                  # 自然柔光 / 霓虹灯光 / 黄昏逆光
  画幅: ""                  # 16:9 或 9:16
  分辨率: "1920x1080"       # 默认 16:9 横屏
  统一风格前缀: ""          # 附加到每个生成提示词开头
  负面提示词: ""            # 统一排除项

# ── 音频设置 ──
音频设置:
  视频内置音频: true         # 始终开启，Seedance 生成环境音效和背景音
  禁止人物语音: true         # 图生视频提示词中要求人物不产生说话声音，角色语音由 Stage 5 TTS 独立控制
```

## 视频类型预设

视频类型决定管线启用哪些 Stage：

| 视频类型 | 启用 Stage | 跳过 Stage |
|---------|-----------|-----------|
| 短剧 | 0-10（全部） | 无 |
| 宣传片/广告 | 0,1,3,4,7,10 | 2,8（5,6,9 可选） |
| 创意短视频 | 0,1,3,4,6,7,10 | 2,5,8,9 |
| 教程 | 0,1,3,4,5,7,9,10 | 2,6,8 |

---

## 冻结规则

用户确认 story.yaml 后：
- **角色 DNA** 和 **风格锁定参数** 在 Stage 1-10 中 **不再变更**
- 如需修改，必须回到 Stage 0 重新确认，并重跑所有受影响的下游阶段
- `统一风格前缀` 是逐字冻结的，下游阶段不可改写

## 输入模式

Stage 0 支持两种输入模式：

### 模式 A：文档快速通道（推荐）

用户提供剧本或创意文档（`.docx`/`.md`/`.pdf`），智能体自动解析并预填 story.yaml。

**支持格式**：
- `.md` / `.txt` — 直接读取（Claude Code Read 工具原生支持）
- `.pdf` — Claude Code Read 工具原生支持 PDF 读取
- `.docx` — 使用文档解析脚本提取文本：
  ```bash
  python ~/.config/opencode/skills/video-s0-creative-planning/scripts/doc_parser.py \
    --input 剧本.docx
  ```

**文档→story.yaml 字段映射**：

智能体读取文档全文后，按以下规则提取信息：

| story.yaml 字段 | 从文档中提取的信息 | 提取难度 |
|-----------------|-------------------|---------|
| `视频类型` | 根据文档性质推断（剧本→短剧，产品介绍→宣传片/广告） | 高置信 |
| `类型风格` | 根据题材推断（办公室→都市，武侠→古装） | 高置信 |
| `主线内容` | 提取故事梗概/剧情大纲/产品卖点（3-5 句） | 高置信 |
| `目标时长` | 根据文本长度和场景数估算 | 需确认 |
| `角色` | 提取人物名称、性别、年龄、外貌描述等 | 部分提取 |
| `世界观与场景设定` | 提取时代、地点、主要场景描述 | 部分提取 |
| `风格锁定` | 根据题材推荐，需用户确认 | 需确认 |
| `音频设置` | 默认值（视频内置音频=true，禁止人物语音=true） | 默认 |

**快速通道流程**：

1. 用户提供文档路径或将文档内容粘贴到对话中
2. 智能体读取/解析文档全文
3. 基于文档内容生成 story.yaml **草稿**，尽可能填充所有字段
4. 向用户展示草稿，标注"**从文档提取**"和"**需要补充/确认**"的字段
5. 用户补充缺失字段（如画幅、风格偏好等）
6. 生成 `统一风格前缀`（必须英文）和 `负面提示词`（必须英文）
7. 确认音频策略
8. 用户确认冻结 → 输出最终 story.yaml

**LLM 提取指令模板**：

读取文档后，使用以下思考框架从文档中提取 story.yaml 各字段：

```
请分析以下剧本/文档内容，提取以下信息用于 AI 视频制作：

1. **视频类型**：这是什么类型的视频？（短剧/宣传片/教程/广告/创意短视频）
2. **类型风格**：题材风格是什么？（都市/古装/科幻/悬疑等）
3. **主线内容**：用 3-5 句话概括核心剧情/卖点
4. **角色清单**：列出所有主要角色，包括姓名、性别、年龄段、外貌描述、服装、标志配饰
5. **场景清单**：列出所有出现的场景/地点，描述环境氛围（光线、色温、情绪）
6. **台词/旁白**：提取每个场景的对白或旁白文本
7. **情绪曲线**：整体情绪走向（如 紧张→高潮→释然）

文档内容：
{document_text}
```

### 模式 B：交互式对话（原有流程）

无文档时，通过多轮对话逐步收集所有信息。

## 对话收集流程

1. **检测文档输入**：如果用户提供了文档路径（.docx/.md/.pdf 后缀）或在消息中 @引用了文件，走模式 A 快速通道；否则走模式 B
2. 询问视频类型和目标时长
3. 收集故事主线（3-5 句概括）
4. 确认角色数量，逐一填写 DNA 卡片
5. 确认主要场景列表及氛围
6. 协商画面风格、色调、光照
7. 生成 `统一风格前缀`（英文，用于 Seedream/Seedance API）
8. 生成 `负面提示词`（英文）
9. 确认音频策略：告知用户"视频会自动生成环境音效（脚步声、雨声等），角色语音将由后续 TTS 阶段独立添加以保证音色一致，视频中人物不会产生说话的声音"
10. 输出完整 story.yaml，请用户确认并冻结
