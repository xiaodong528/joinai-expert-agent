# 分镜脚本验证 (Storyboard Validation)

Stage 1→2/3 之间的质量门控。读取 `storyboard.yaml` 和 `story.yaml`，逐项检查结构完整性和规则合规性，输出结构化验证报告。仅当总评 PASS 时方可进入后续 Stage。

---

## 输入

| 文件 | 路径 | 用途 |
|------|------|------|
| storyboard.yaml | `Video-Producer-output/{project_id}/scripts/storyboard.yaml` | 验证目标 |
| story.yaml | `Video-Producer-output/{project_id}/scripts/story.yaml` | 交叉验证：风格锁定、角色 DNA、目标时长 |

---

## 检查清单（4 类 13 项）

### 结构检查

1. **YAML 语法**: 文件可被标准 YAML 解析器解析
2. **必填顶层字段**: `元信息`、`风格锁定`、`分镜列表` 均存在
3. **场景数量一致**: `元信息.场景总数` = `分镜列表` 实际长度
4. **场景数量正确**: 实际场景数 = `ceil(目标总时长(秒) / 单镜时长(秒))`
5. **编号连续**: `编号` 从 1 开始连续递增，无跳号无重复

### 每场景字段检查

6. **必填字段非空**: 编号、时长、景别、情绪、文生图提示词、图生视频提示词
7. **时长合规**: 每个分镜时长 <= 12 秒
8. **景别有效值**: 必须为以下之一：`远景` / `全景` / `中景` / `近景` / `特写`

### 规则合规检查

9. **R2 风格前缀**: 每个 `文生图提示词` 以 `风格锁定.统一风格前缀` 的前 3 个关键词开头
10. **R5 静态画面**: `文生图提示词` 中不包含镜头运动词（pan, tilt, zoom, tracking, dolly, orbit, push in, pull out, camera moves, sweeping）
11. **R5 动态指令**: `图生视频提示词` 中包含至少一个镜头运动或动作描述
12. **R3 时间连贯**: 相邻分镜的光线/天气/时间描述不矛盾（标注可疑处供人工确认）

### 角色 DNA 检查（仅当 story.yaml 有角色时）

13. **R1 DNA 完整**: 每个包含角色的 `文生图提示词` 包含角色外貌的核心关键词（发色、服装、标志配饰），与 story.yaml 角色 DNA 卡片交叉比对

### 末帧图提示词检查（可选，仅当字段存在时）

14. **末帧 R2 风格前缀**: `末帧图提示词` 以 `风格锁定.统一风格前缀` 开头
15. **末帧 R5 静态画面**: `末帧图提示词` 中不包含镜头运动词
16. **末帧 R1 DNA 完整**: `末帧图提示词` 包含角色外貌核心关键词（如有角色）

> 如果 story.yaml 中 `角色` 列表为空（如创意短视频），此类检查标记为 N/A。

---

## 交叉验证（与 story.yaml 比对）

除 13 项主检查外，还需交叉验证以下字段一致性：

| storyboard.yaml 字段 | story.yaml 字段 | 检查方式 |
|----------------------|-----------------|---------|
| `风格锁定.统一风格前缀` | `风格锁定.统一风格前缀` | 字符串完全一致 |
| `风格锁定.负面提示词` | `风格锁定.负面提示词` | 字符串完全一致 |
| `元信息.画幅` | `目标画幅` | 值一致 |

## 修复指引

| 失败类型 | 建议操作 |
|---------|---------|
| YAML 语法错误 | 检查缩进和特殊字符，修复 YAML 格式 |
| 场景数量不匹配 | 重新执行 Stage 1，确认 `ceil(目标时长/单镜时长)` |
| 风格前缀不一致 | 手动将 `统一风格前缀` 复制到各提示词开头 |
| 文生图含运动描述 | 将运动描述移到 `图生视频提示词`，文生图只保留静态画面 |
| 图生视频缺少运动 | 添加至少一个镜头运动指令（参考 S1 Skill 的运动词汇表） |
| 角色 DNA 缺失 | 从 story.yaml 角色卡片复制完整外貌描述到提示词 |
| 时间连贯性存疑 | 人工审查相邻分镜的光线/天气描述，确保时间单调递进 |
| 风格锁定交叉不一致 | 从 story.yaml 原样复制到 storyboard.yaml 的 `风格锁定` |

**自动修复策略**: 如 FAIL 项 <= 3 且均为可自动修正类（风格前缀、DNA 缺失），直接修正 storyboard.yaml 并重新验证。如 FAIL 项 > 3 或涉及结构性问题，重新执行 Stage 1。
