## 2. 各阶段详细评估

### Stage 0-1 & 10: LLM — 剧本、分镜脚本与质量审查（必选）

#### 选型决策

**统一使用 GasTown 底层 provider（OpenCode）默认模型**

Stage 0（剧本生成）、Stage 1（分镜脚本）、Stage 10（质量审查）的文本生成任务统一由 GasTown 的底层 LLM provider 处理，不单独选型。

- 模型由 OpenCode provider 配置决定，跟随 GasTown 全局设置
- 成本由 GasTown 运行时统一承担，不计入本管线模型成本
- 所有 LLM 调用通过 Mayor/Polecat 的标准接口发起，无需额外 API Key
- 要求：provider 模型需支持结构化输出（JSON mode）和足够的上下文窗口（建议 128K+）

### Stage 4: 视频生成 — 图生视频（必选）

#### 综合对比

| 模型 | 8s/720p 成本 | 首尾帧 | 静音输出 | 时长控制 | 并发 | 推荐 |
|------|-------------|--------|---------|---------|------|------|
| **Seedance 1.5 Pro** | **~$0.14** | ✅ 完全支持 | ✅ `generate_audio` 可控 | 4/8/12s | 10 | 首选 |
| Wan 2.2 kf2v | ~$0.11 (5s) | ✅ 专有模型 | ✅ 天然静音 | 固定5s | 1 | 次选 |
| 可灵 1.6 Pro | ~$0.45 | ✅ 支持 | ✅ `enable_audio=false` | 5/10s | 5 QPS | 次选 |
| Runway Gen-4 Turbo | $0.40 | ❌ 无尾帧 | ✅ 天然静音 | 2-10s | — | 备选 |
| Sora 2 | $0.80 | ❌ 无尾帧 | ❓ 不明确 | 4/8/12s | — | 不推荐 |

#### 选型决策

**首选: 即梦 Seedance 1.5 Pro（火山方舟）**
- `generate_audio=true` 始终启用，提示词末尾附加 "ambient sounds only" 避免人声干扰
- 首尾帧通过 `role` 字段显式标注（`"role": "first_frame"` / `"role": "last_frame"`），确保场景连续性
- `duration` 有效范围 4–12s，默认设为 **12s** 以获取最大生成时长
- 摄像机运动指令最丰富（pan/tilt/zoom/tracking）
- 并发 10 个任务，适合批量生产
- **10场景总成本仅 ~$1.40（¥10.08）**

**容灾备份: Wan 2.2 kf2v-flash（阿里云百炼）**
- kf2v 模型天然静音，无需设置参数
- 首尾帧是核心设计，稳定性最高
- 成本最低（~¥0.10/秒）
- 限制：固定 5 秒输出，并发仅 1 个处理中任务

**高并发扩容: 可灵 1.6 Pro**
- 若 Seedance 10 并发不够，可作为溢出队列
- `enable_audio=false` 默认值，与尾帧兼容

#### API 调用示例

```python
# Seedance 1.5 Pro — 首尾帧（role 标注）+ 启用环境音
import volcenginesdkarkruntime

client = volcenginesdkarkruntime.Ark(api_key="ARK_API_KEY")
response = client.content_generation.tasks.create(
    model="doubao-seedance-1-5-pro-251215",
    content=[
        {"type": "image_url", "image_url": {"url": "https://cdn.example.com/first.jpg"}, "role": "first_frame"},
        {"type": "image_url", "image_url": {"url": "https://cdn.example.com/last.jpg"}, "role": "last_frame"},

task_id = response.id

# Wan 2.2 kf2v — 容灾备份（天然静音）
from dashscope import VideoSynthesis

        "first_frame_url": "https://cdn.example.com/start.jpg",
        "last_frame_url": "https://cdn.example.com/end.jpg",
        "prompt": "平稳过渡，保持角色一致性"
    },
    parameters={"resolution": "720P", "prompt_extend": True}
```

### Stage 5: TTS 语音合成（可选）

#### 综合对比

| 模型 | 单价 | 声音克隆 | 情感控制 | 中文质量 | 跨片段一致性 |
|------|------|---------|---------|---------|------------|
| **火山引擎豆包 TTS 2.0** | ¥3/万字符 | ✅ 声音复刻 | ✅ 情感标签 | ★★★★★ | voice_id 固定 |
| CosyVoice v3.5-plus | ¥2/万字符 | ✅ 零样本克隆(免费) | ✅ 指令控制 | ★★★★☆ | voice_id 固定 |
| Fish Speech 1.5 | ~¥0.85/万字符 | ✅ 含在定价 | ⚠️ 有限 | ★★★☆☆ | 参考音频绑定 |
| ElevenLabs v3 | ~¥1.2-2.2/万字符 | ✅ Pro克隆 | ✅ 最强 | ★★★☆☆ | voice_id 固定 |

#### 选型决策

**首选: 火山引擎豆包语音合成 2.0 大模型（seed-tts-2.0）**
- 大模型级语音质量，拟人感和情感演绎显著优于 1.0
- V3 异步长文本 API：`POST https://openspeech.bytedance.com/api/v3/tts/submit`
- 认证方式：`X-Api-App-Id` + `X-Api-Access-Key` + `X-Api-Resource-Id: seed-tts-2.0`
- `enable_timestamp: true` 返回句级+字级时间戳（秒 float），直接生成 SRT 字幕
- 33 个 2.0 专属音色（`_uranus_bigtts` 后缀），默认推荐 `zh_male_ruyayichen_uranus_bigtts`（儒雅逸辰 2.0，视频配音）
- **单片成本: ¥0.18**（600字台词 ≈ 0.06万字符）

```python
# 火山引擎豆包语音合成 2.0 大模型 — V3 异步长文本 API
import requests, uuid

headers = {
    "Content-Type": "application/json",
    "X-Api-App-Id": TTS_APP_ID,
    "X-Api-Access-Key": TTS_TOKEN,
    "X-Api-Resource-Id": "seed-tts-2.0",
    "X-Api-Request-Id": str(uuid.uuid4()),
}

# 1. 提交合成任务
resp = requests.post(
    "https://openspeech.bytedance.com/api/v3/tts/submit",
    headers=headers,
    json={
        "user": {"uid": "pipeline"},
        "unique_id": str(uuid.uuid4()),
        "req_params": {
            "text": "今天天气真好，我们一起去郊游吧！",
            "speaker": "zh_male_ruyayichen_uranus_bigtts",  # 儒雅逸辰 2.0
            "audio_params": {
                "format": "mp3",
                "sample_rate": 24000,
                "enable_timestamp": True,  # 返回字幕时间戳
            },
        },
    },
)
task_id = resp.json()["data"]["task_id"]

# 2. 轮询（task_status: 1=Running, 2=Success, 3=Failure）
result = requests.post(
    "https://openspeech.bytedance.com/api/v3/tts/query",
    headers=headers,
    json={"task_id": task_id},
).json()["data"]
# result -> {"audio_url": "...", "sentences": [{"text": "...", "startTime": 0.28, "endTime": 6.07}]}
```

### Stage 6: BGM 音乐生成（可选，推荐）

#### 综合对比

| 模型 | 单价 | Instrumental | 时长控制 | 风格控制 | API 稳定性 |
|------|------|-------------|---------|---------|----------|
| **MiniMax Music 2.5+** | $0.15/首 | ✅ `is_instrumental:true` | ⚠️ prompt引导 | ✅ | ✅ 官方API |
| Mureka V8 | ~$0.12-0.15/分钟 | ✅ 器乐模式 | ⚠️ 有限 | ✅ 200+风格 | ✅ 官方API |
| ACE-Step 1.5 | 免费(开源) | ✅ | ✅ 10s-10min | ✅ 1000+风格 | 需自建 |
| Suno v5 | ~$0.04/首(第三方) | ✅ | ✅ 8min | ✅ | ⚠️ 无官方API |

#### 选型决策

**首选: MiniMax Music 2.5+**
- $0.15/首（最长5分钟），官方 REST API 稳定
- `is_instrumental: true` 原生支持纯音乐 BGM
- 中国站端点：`https://api.minimaxi.com/v1/music_generation`（注意域名是 `minimaxi.com`）
- API 实际为**同步模式**：直接返回 hex 编码音频数据，非异步轮询；建议超时 >= 300s
- **单片成本: ¥1.09**

```python
# MiniMax Music 2.5+ — 同步模式（直接返回 hex 编码音频）
import requests

resp = requests.post(
    "https://api.minimaxi.com/v1/music_generation",  # 中国站 minimaxi.com
    json={
        "model": "music-2.5+",
        "prompt": "Suspenseful cinematic BGM, orchestral, tense atmosphere",
        "is_instrumental": True,
        "audio_setting": {"format": "mp3", "sample_rate": 44100}
    },
    headers={"Authorization": f"Bearer {MINIMAX_KEY}"},
    timeout=300  # 同步接口，生成耗时较长，建议 >= 300s
)
# 响应中 audio_hex 为 hex 编码音频，需 bytes.fromhex() 解码后写入文件
audio_hex = resp.json()["data"]["audio_hex"]
with open("bgm.mp3", "wb") as f:
    f.write(bytes.fromhex(audio_hex))
```

## 4. 推荐工具链组合

### 方案 A: 纯 SaaS 全家桶（零 GPU，推荐起步）

```
GasTown/OpenCode (剧本+分镜+QA)
  → 即梦 Seedream 4.0 (角色锚定+关键帧)
  → 即梦 Seedance 1.5 Pro (图生视频, generate_audio=true)
  → 火山引擎豆包 TTS 2.0 (语音合成)
  → MiniMax Music 2.5+ (BGM)
  → 可灵 Lip-Sync API (对口型)
  → 火山引擎豆包 ASR 2.0 + FFmpeg (字幕+拼接)
  → ffprobe (QA 技术指标验证)
```

**总成本: ~¥8~14/片（不含 LLM）** | **API Key: 4个** | **GPU: 不需要**

### 实施封装方式

方案 A 的每个环节将封装为 **混合式 OpenCode Skill**：核心 API 调用逻辑固化为 `scripts/*.py` 脚本（统一 auth/retry/poll 模式），灵活部分（prompt 生成、质量判断、备选切换）以指令式写入 Skill。全部 11 个 Skill 由 `video-generation-orchestrator` Agent 编排调用，MCP 仅用于复杂感知类能力（视觉理解/网络搜索/视频分析），API 调用不走 MCP。

### 方案 B: SaaS + 本地推理混合（角色一致性增强）

```
GasTown/OpenCode (剧本+分镜+QA)
  → ComfyUI + FLUX.1 Dev + IP-Adapter/LoRA (角色锚定)
```
